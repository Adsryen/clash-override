# 性能测试工具

用于测试 `global_script.js` 处理 YAML 配置文件的性能。

## 文件说明

- `test_simple.js` - 简化版测试脚本，快速查看处理速度
- `test_performance.js` - 详细版测试脚本，包含完整统计信息
- `test.yaml` - 示例配置文件（可替换为你自己的配置）

## 使用方法

### 1. 安装依赖

```bash
npm install js-yaml
```

### 2. 运行简单测试

```bash
cd test
node test_simple.js test.yaml
```

输出示例：
```
测试文件: test.yaml
节点数量: 50

第 1 次: 15 ms
第 2 次: 12 ms
...
平均耗时: 13.50 ms
最快: 12 ms
最慢: 18 ms
```

### 3. 运行详细测试

```bash
cd test
node test_performance.js test.yaml 10
```

参数说明：
- 第一个参数：YAML 配置文件路径（默认 `test.yaml`）
- 第二个参数：测试次数（默认 10 次）

输出示例：
```
============================================================
开始性能测试
============================================================
代理节点数量: 50
测试次数: 10
------------------------------------------------------------
预热运行中...
正式测试中...

第 1 次: 15.23 ms
第 2 次: 12.45 ms
...

============================================================
测试结果统计
============================================================
平均耗时: 13.50 ms
最快耗时: 12.45 ms
最慢耗时: 18.67 ms
中位数: 13.20 ms
------------------------------------------------------------

处理后的配置信息:
策略组数量: 25
规则数量: 150
规则提供者数量: 8
============================================================

处理后的配置已保存到: test_output.yaml
```

## 测试你自己的配置

将你的 YAML 配置文件放到 test 文件夹，然后运行：

```bash
node test_simple.js 你的配置.yaml
```

## 性能优化对比

如果你想对比优化前后的性能差异：

1. 备份原始的 `global_script.js`
2. 运行测试记录结果
3. 应用优化后再次测试
4. 对比平均耗时的变化

## 注意事项

- 测试会深拷贝配置对象，不会修改原始文件
- 第一次运行会进行预热，不计入统计
- 建议测试次数设置为 10-20 次以获得稳定结果
- 节点数量越多，性能差异越明显
